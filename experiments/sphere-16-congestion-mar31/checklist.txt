sphere-16-congestion-mar31 — Network Congestion Tolerance Checklist (8 total)
=========================================================================
Goal: Compare AsyncMoE and SGLang EP16 tolerance to trace-driven network
interference on gptoss-120b sharegpt workload (balanced gate profile).

Uses the interference_gen/ tool to replay real cloud noise traces on
the cluster's RoCE links while running the serving benchmark.


CLUSTER & SYSTEM SETUP
=========================================================================
  Cluster:     8 nodes × 2 L40S = 16 GPUs (Sphere-16)
  Model:       gptoss_120b (36L, 128E, top-4, bf16)
  Gate:        balanced variant only (sharegpt)
  AsyncMoE:    conda disag12 | cwd ~/DisagMoE     | MEM_FRAC=0.98
  SGLang:      conda sglang-fp | cwd ~/sglang-fake-prefill | MEM_FRAC=0.77
  Interference: ~/DisagMoE/interference_gen/

  Run AsyncMoE first, then SGLang. Never simultaneously.


BENCHMARK PARAMETERS (same as itl-tput sweep)
=========================================================================
  Rate:            400 rps
  Duration:        100 seconds
  Total requests:  40000 (asyncmoe: rate×time, sglang: BENCH_NUM_PROMPTS)
  Distribution:    dataset (sharegpt_lengths.npy)
  Input bounds:    min=256, max=512
  Output bounds:   min=256, max=512
  Max context len: 2048
  SGLang:          BENCH_DISABLE_STREAM=1 (mandatory)
  Retries:         AsyncMoE max 3, SGLang max 2

RUN PROCEDURE (per experiment)
=========================================================================

  1. PRE-FLIGHT: kill any previous interference + serving processes
     (see itl-tput-sweep-guide.md for the full kill procedure)

  2. START INTERFERENCE (in a separate tmux session):
     tmux new-session -d -s interfere
     tmux send-keys -t interfere "<ring or 2-node command from above>" Enter

  3. WAIT 2 MINUTES for calibration to complete.
     Verify calibration done:
       grep "Calibration complete\|Starting interference" <interfere tmux output>

  4. LAUNCH BENCHMARK (in another tmux session):
     Follow the exact same per-run procedure from itl-tput-sweep-guide.md
     for AsyncMoE or SGLang, with RATE=400, TIME=100, and PROFILE=sharegpt_balanced.

  5. WAIT FOR BENCHMARK to complete (monitor as usual).

  6. STOP INTERFERENCE:
     tmux send-keys -t interfere C-c
     # or use the --stop command
     tmux kill-session -t interfere

  7. RENAME output directory to include trace tag:
      mv .../asyncmoe-sharegpt_balanced \
         .../asyncmoe-sharegpt_balanced-400rps-aws-ring
      (or -azure-ring, -aws-single-link, -no-interference)

  8. COLLECT METRICS: run parse_detokenizer_logs.py on the experiment dir.

WARNINGS — FREQUENTLY ENCOUNTERED ISSUES
=========================================================================

  1. AFTER CLUSTER REBOOT: gdrdrv + ldconfig must be restored
     Reboots wipe the gdrdrv kernel module and ld.so.conf entries. On EVERY
     node, run:
       cd ~/gdrcopy && sudo bash ./insmod.sh
       sudo ldconfig
     Then verify: python -c "import disagmoe_c"  (disag12 env)
                  python -c "import sglang"       (sglang-fp env)
     sgpu7 in particular was never fully set up — it may need apt packages
     (libzmq3-dev, libnccl2, etc.), miniconda rsync, and make pip from scratch.

  2. SGLANG: ports 25000-25005 and 30000 MUST be freed before every run
     SGLang binds to these ports and zombie processes hold them after kills.
     Always run fuser -k on ALL ports on ALL nodes as pre-flight:
       for n in sgpu0 sgpu2 sgpu3 sgpu4 sgpu6 sgpu7 sgpu8 sgpu9; do
         ssh $n "pkill -9 -f sglang; fuser -k 25000/tcp 25001/tcp 25002/tcp \
           25003/tcp 25004/tcp 25005/tcp 30000/tcp" &
       done; wait; sleep 8
     Verify with: fuser 25000/tcp 30000/tcp  (must produce no output)

  3. SGLANG: --disable-stream is MANDATORY
     Without it, sustained request rates cause HTTP connection exhaustion
     (ConnectionResetError / Cannot assign requested address). This means
     SGLang ITL from bench_result.json will be zero — use the server-side
     detokenizer logs (logs/server_head.log) for ITL data instead.

  4. SGLANG: Gloo crash in prepare_mlp_sync_batch_raw
     torch.distributed.all_gather_into_tensor can throw "Connection closed
     by peer" from a specific node. This was caused by flaky hardware and
     resolved by a full cluster reboot. If it recurs, reboot the cluster.

  5. ASYNCMOE: check detokenizer tput logs as health signal
     AsyncMoE detokenizer logs "token throughput: XX.XXk tokens/s" every
     ~10 seconds. If no log line appears for >60 seconds, the system is
     hung — kill the server and retry. This is the ONLY reliable way to
     detect hangs; the benchmark curl may appear to still be running.

  6. EVAL SCRIPT OUTPUT DIRECTORY NAMING
     Both eval scripts write to a fixed default directory name (e.g.
     asyncmoe-sharegpt_balanced or sglang_ep16-sharegpt_balanced).
     You MUST rename it after each run (the && mv in the tmux command).
     If the eval script crashes, the rename doesn't happen — rename manually
     before the next run, or the next run will overwrite the data.

  7. INTERFERENCE + BENCHMARK TIMING
     Start interference FIRST. Wait the full 2 minutes for calibration to
     finish before launching the benchmark. If you start the benchmark
     during calibration, the first ~2 min of benchmark data will be under
     no interference (misleading results). Verify calibration is done:
       grep "Calibration complete\|Starting interference" <interfere log>

  8. INTERFERENCE CLEANUP
     Always stop interference AFTER the benchmark finishes. If you kill
     interference mid-benchmark, the results will mix interference and
     no-interference periods. Use the --stop command or Ctrl-C in the
     interfere tmux session.

  9. CODE SYNC AFTER CHANGES
     No NFS on this cluster. After ANY code change on sgpu0, rsync to
     all workers before running. If C++ changed, rebuild (make pip) on
     every node. Python-only changes need rsync only.


POST-EXPERIMENT DATA PROCESSING
=========================================================================

  Parser script: ~/DisagMoE/experiments/evaluations/parse_detokenizer_logs.py

  This script reads server logs from both AsyncMoE (server.log) and SGLang
  (logs/server_head.log), extracts per-second throughput and ITL metrics,
  and computes throughput-weighted aggregates. The weighting ensures that
  steady-state (high-tput) periods dominate over ramp-up and drain phases.

  Usage:
    conda activate disag12
    cd ~/DisagMoE
    python experiments/evaluations/parse_detokenizer_logs.py \
      experiments/sphere-16-congestion-mar31/ \
      --csv experiments/sphere-16-congestion-mar31/metrics.csv

  NOTE: The parser auto-discovers asyncmoe-gptoss-results/ and
  sglang-gptoss-results/ subdirectories. Result dirs must follow the
  naming pattern <system>-<workload>-<rate>rps (e.g. asyncmoe-sharegpt_balanced-300rps).
  The congestion trace tag (e.g. -aws-ring) will be included in the directory
  name but the parser only needs the system/workload/rate prefix to match.
  You may need to adjust parse_run_name() regex if using non-standard suffixes.

  Output CSV columns:
    system, workload, rate_rps, num_samples, peak_tput, avg_tput,
    weighted_itl_mean_ms, weighted_itl_median_ms, weighted_itl_p99_ms

  For full documentation, see: experiments/evaluations/itl-tput-sweep-guide.md
