OPTIMIZATION INVENTORY
=========================================================================

  1. GDRCopy optimization
     Controls host↔GPU copies via GDRCopy pinned memory instead of cudaMemcpy.
     Toggle: disagmoe/utils/gdr_context.py line 5: use_gdrcopy_optimization = True/False
     IMPORTANT: No CLI flag. Must edit the file and sync to ALL worker nodes (no NFS).

  2. Compute-communication overlap (overlap scheduler)
     The engine main loop has two variants:
       - single_module_loop_overlap (default) — pipelines send with next compute
       - single_module_loop          — synchronous: compute → wait → send → next
     Toggle: disagmoe/frontend/engine.py line 674:
       Overlap ON:  self.loop_thread = Thread(target=self.single_module_loop_overlap)
       Overlap OFF: self.loop_thread = Thread(target=self.single_module_loop)
     IMPORTANT: No CLI flag. Must edit the file and sync to ALL worker nodes (no NFS).

  3. Defrag scheduler lookahead/lookback
     Default: lookahead=4, lookback=4 (full defrag)
     MBFS: lookahead=1, lookback=0 (minimum batch first scheduling)
     Toggle: env vars DEFRAG_LOOKAHEAD_STEPS / DEFRAG_LOOKBACK_STEPS
