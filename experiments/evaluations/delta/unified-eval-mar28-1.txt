## RUNNING EACH SCRIPT

For each of the 3 eval scripts above, follow the same pattern:

  1. Get a shell on the head compute node (inside the SLURM allocation):

       srun --jobid=17003239 --nodelist=gpua003 --overlap --pty bash

     If the job ID has changed, find it first:
       squeue -u $USER

  2. Activate the correct conda env and source env if needed:

       AsyncMoE scripts → source ~/miniconda3/etc/profile.d/conda.sh
                          source ~/DisagMoE/experiments/scripts/delta/env.sh
                          (initializes conda, activates env 'amoe', sets LD_LIBRARY_PATH)
       SGLang scripts   → source ~/miniconda3/etc/profile.d/conda.sh && conda activate sglang

  3. cd to the correct repo root:

       AsyncMoE scripts → cd ~/DisagMoE
       SGLang scripts   → cd ~/sglang

  4. Verify no lingering processes from the previous run:

       ps aux | grep -E 'ray|server\.py|sglang|benchmark' | grep -v grep

     Kill any that remain. For SGLang also check tmux sessions:
       tmux ls 2>/dev/null

  5. Launch in background, piping stdout/stderr to a log file:

       OUTDIR=~/unified-eval-mar28-1
       TAG=<short name, e.g. asyncmoe-gptoss, asyncmoe-glm45air, sglang-gptoss>
       RESULTS=$OUTDIR/${TAG}          # results dir: {system}-{model}/{dataset}
       mkdir -p "$RESULTS"

       # SLURM_JOB_ID / SLURM_JOB_NODELIST are set automatically inside an
       # srun --pty shell. HEAD_IP is auto-detected via hostname -I.
       # Set them explicitly only when launching from a non-SLURM shell.
       SLURM_JOB_ID=<JOBID> SLURM_JOB_NODELIST=<NODELIST> HEAD_IP=<HEAD_IP> \
           nohup bash <script> "$RESULTS" > "$OUTDIR/${TAG}.log" 2>&1 &
       echo $! > "$OUTDIR/${TAG}.pid"

  6. Monitor until the script prints its completion message, then move to the next.


## RUNNING A SINGLE EXPERIMENT

All eval scripts support --list and --only options for selective execution:

  List available experiments (prints index + name, then exits):
       bash <script> "$RESULTS" --list

  Run specific experiments by index (1-based):
       bash <script> "$RESULTS" --only 1
       bash <script> "$RESULTS" --only 1,3

  Run by name substring (matches against the run name):
       bash <script> "$RESULTS" --only sharegpt_regular
       bash <script> "$RESULTS" --only sharegpt

  For SGLang scripts, filter by server profile:
       bash <script> "$RESULTS" --only ep16
       bash <script> "$RESULTS" --only ep16-sharegpt_regular

  Omitting --only runs all experiments (default behavior, unchanged).


## MONITORING

Check each running script at least every 6 minutes:

  1. Process alive?   ps -p $(cat <pid_file>)
  2. Progress?        tail -80 <log_file>
     Healthy: timestamps advancing, "SUCCESS", "Attempt N/3", "OOM detected", or at least some progress
  3. Anomalies: no output >15 min (hung), ALL experiments failing systematically,
     process dead unexpectedly.

Single experiment failing after retries is normal — the script moves on, but systematic failing need to be noted.


## INTERVENTION POLICY

## LESSONS LEARNED (from unified-eval-mar28-1 run)

### AsyncMoE

- SLURM env vars (SLURM_JOB_ID, SLURM_JOB_NODELIST) and HEAD_IP must be
  explicitly set when launching nohup from a non-SLURM shell. Inside srun
  --pty bash they are set automatically.
- MEM_FRAC=0.92 works for both gptoss and glm45air on ep16.
- gsm8k experiments require datasets/gsm8k_lengths.npy; sharegpt experiments
  require datasets/sharegpt_lengths.npy. Both eval scripts now auto-select.
- The MINICONDA path moved from /projects/bgro/spark36/miniconda3 to ~/miniconda3.
  config.sh updated to use $HOME/miniconda3.

### SGLang

- --disable-custom-all-reduce is required on ALL EP profiles (ep16, ep8).
  Without it, CUDA graph capture hangs indefinitely (Gloo fallback deadlock).
  evallib/server.sh updated; was previously only set on pp4tp4.

- MEM_FRAC is profile-dependent:
    ep16, ep16_limited, pp4tp4 → 0.65 (confirmed; 0.70 deadlocks with
                                       mooncake-nccl during capture)
    ep8                        → 0.85 minimum (8-GPU sharding, ~30 GB
                                       weights/GPU; 0.75 still OOM)
  config.sh updated: MEM_FRAC=0.65, EP8_MEM_FRAC=0.85.
  server.sh automatically applies EP8_MEM_FRAC for ep8 profile.

- CUDA_LAUNCH_BLOCKING=1 deadlocks with mooncake-nccl (async dependency).
  Do not use for production runs.

- Use MODEL_PATH="lmsys/gpt-oss-120b-bf16" (HuggingFace ID). Local copies
  at /projects/bgro/spark36/models/ may be stale. Works with --load-format dummy.

- Balanced gate profiles live in gptosss_balanced_output/ (triple-s), not
  balanced_output/. config_gptoss.sh corrected.

- Always monitor server_head.log directly, not just the [main] eval output.
  OOM errors and worker crashes only appear in server_head.log / server_wN.log.

- Killing a stuck eval: kill $PID only removes the nohup wrapper. Always
  pkill -9 -f sglang on all nodes + tmux kill-session for sglang-head/wN.
