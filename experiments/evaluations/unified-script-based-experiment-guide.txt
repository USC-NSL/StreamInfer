## RUNNING EACH SCRIPT

For each of the 4 eval scripts above, follow the same pattern:

  1. SSH into sgpu0.
  2. Activate the correct conda env:
       AsyncMoE scripts → conda activate disag12
       SGLang scripts   → conda activate sglang-fp
  3. cd to the correct repo root:
       AsyncMoE scripts → cd ~/DisagMoE
       SGLang scripts   → cd ~/sglang-fake-prefill
  4. Verify no lingering processes from the previous run:
       ps aux | grep -E 'ray|server\.py|sglang|benchmark' | grep -v grep
     Kill any that remain.
  5. Launch in background, piping stdout/stderr to a log file:

       OUTDIR=~/DisagMoE/experiments/sphere-16-unified-eval
       TAG=<short name, e.g. asyncmoe-gptoss, sglang-glm45air>
       RESULTS=$OUTDIR/${TAG}-results
       mkdir -p "$RESULTS"

       nohup bash <script> "$RESULTS" > "$OUTDIR/${TAG}.log" 2>&1 &
       echo $! > "$OUTDIR/${TAG}.pid"

  6. Monitor until the script prints its completion message, then move to the next.


## RUNNING A SINGLE EXPERIMENT

All 4 eval scripts support --list and --only options for selective execution:

  List available experiments (prints index + name, then exits):
       bash <script> "$RESULTS" --list

  Run specific experiments by index (1-based):
       bash <script> "$RESULTS" --only 1
       bash <script> "$RESULTS" --only 1,3

  Run by name substring (matches against the run name):
       bash <script> "$RESULTS" --only sharegpt_regular
       bash <script> "$RESULTS" --only sharegpt

  For SGLang scripts, filter by server profile:
       bash <script> "$RESULTS" --only pp8tp2
       bash <script> "$RESULTS" --only ep16-sharegpt_regular

  Omitting --only runs all experiments (default behavior, unchanged).


## MONITORING

Check each running script at least every 6 minutes:

  1. Process alive?   ps -p $(cat <pid_file>)
  2. Progress?        tail -80 <log_file>
     Healthy: timestamps advancing, "SUCCESS", "Attempt N/3", "OOM detected", or at least some progress
  3. Anomalies: no output >15 min (hung), ALL experiments failing systematically,
     process dead unexpectedly.

Single experiment failing after retries is normal — the script moves on, but systematic failing need to be noted.
